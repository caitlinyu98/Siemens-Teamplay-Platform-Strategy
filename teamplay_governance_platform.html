Platform layer separation with governance

A unified view of how Siemens resolves its cannibalization fear — three structural layers define what Siemens owns forever, what it keeps selectively, and what it fully opens to partners. The three-lane governance model controls which partners can access which layer, and under what terms.

How to read this diagram
🔒
Layer 1 — Substrate Siemens owns forever. Hardware, fleet data, auth, connectivity. The moat. No partner access.
⚖️
Layer 2 — Sponsor Siemens keeps what's scanner-native. Opens what 3rd parties do better. Selective.
🏪
Layer 3 — Provider Fully open to partners. Siemens governs but does not build. Clinical AI, apps, reporting.
🏛️
Governance layer Sits above all providers. Lane A/B/C classification, SLA standards, cert, billing, ROI.

🏥 Scanner hardware MAGNETOM · SOMATOM · Biograph. 38,000+ installed globally. Moat
📡 Fleet data layer Utilization · uptime · cross-hospital coordination. No one else has this. Moat
⚙️ teamplay platform Auth · billing · API gateway · app lifecycle management.
🔗 Connectivity DICOM · PACS · HL7 · FHIR. The pipe every app needs.

The cannibalization fear is real but misdirected. Siemens is protecting Layer 2 syngo capabilities that third parties will build faster and better regardless. The actual moat is the substrate — 38,000+ installed scanners generating fleet data no competitor can replicate. Opening the provider layer creates network effects. Protecting the substrate creates lock-in. Both compound each other.
